# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Sandbox Action, please email
**security@airut.org** instead of opening a public issue.

Please wait for a fix before publishing details.

## Scope

Before reporting, review the [Security Model](README.md#security-model) section
of the README to understand Sandbox Action's security boundaries. Consider
whether your finding undermines the documented guarantees (container isolation,
network allowlisting, credential masking) or falls outside them (e.g., actions
within allowlisted scope, misconfigured branch protection).

## Response

Airut is an experimental project developed between other commitments. We will
address reports on a best-effort basis but cannot guarantee specific timelines.
